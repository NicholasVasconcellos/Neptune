# Individual Interval Calculation Feature

## Context

When a coach creates a training, they assign a group (team) and set an effort level (N1-N4) per exercise. Currently there's only a single manual interval per exercise. This feature calculates **personalized intervals for each swimmer** in the group based on their best time for the exercise's stroke+distance, divided by the effort level's percentage. This gives coaches instant, per-athlete send-off times.

## Summary of Decisions

- **Stroke**: Add a new Stroke selector (Typeahead from `SWIM_STROKES`) to ExerciseRow
- **Effort defaults**: N1=70%, N2=80%, N3=90%, N4=95% (N1 slowest, N4 near race pace)
- **Display**: Collapsible section below each confirmed exercise row
- **Rounding**: Round UP to nearest 5 seconds
- **Manual interval coexists** independently with individual intervals
- **Settings**: New 5th tab for customizing effort % mappings
- **Unit matching**: Exact match only (yards↔yards, meters↔meters)
- **Effort mappings**: Persisted in Supabase per user

## Step 2: Pure Utility — `utils/intervalCalculation.ts` (NEW)

Standalone, testable module with no UI or network code:

```typescript
calculateInterval(bestTime: number, effortPercentage: number): number
// Returns Math.ceil(bestTime / effortPercentage / 5) * 5
```

Also exports an `AthleteInterval` type and a `calculateAllIntervals()` batch function.

## Step 3: Data Layer — Two New Utility Files

### 3a. `utils/bestTimes.ts` (NEW)
- `fetchBestTimesForTeam(teamId, stroke, distance, distanceUnit)`
- Fetches athletes in team via `getData("Athletes", { "Team ID": teamId })`
- Fetches all user's times via `getData("Times")`
- Client-side: filters by athlete, stroke, distance, unit → picks `Math.min(Time)`
- Returns `{ id, Name, bestTime }[]` — only athletes with a matching time

### 3b. `utils/effortMappings.ts` (NEW)
- `getEffortMappings()` → `Record<string, number>` (fetches from DB, seeds defaults if empty)
- `updateEffortMapping(energySystem, percentage)` → updates single row

### 5d. Interval computation function
`computeIntervalsForExercise(exerciseId, exercise, teamId)`:
1. Gets `effortMappings[exercise.energySystem]`
2. Calls `fetchBestTimesForTeam(teamId, exercise.stroke, distance, unit)`
3. Calls `calculateAllIntervals(athletes, percentage)`
4. Stores result in `individualIntervals[exerciseId]`

### 5e. Trigger on exercise confirm (modify `confirmExercise` at line 357)
After setting `confirmed: true`, if exercise has stroke + energySystem + distance + teamId → call `computeIntervalsForExercise`.

### 5f. Recalculate on teamId/unit/effortMappings change
`useEffect` that recalculates all confirmed qualifying exercises when these dependencies change. Clear intervals when teamId becomes null.

### 5g. Clear intervals on un-confirm
In `updateExercise`, when `field === "confirmed"` and `value === false`, delete that exercise's intervals.

### 5h. Pass to ExerciseRow (line 886-898)
Add `individualIntervals={individualIntervals[ex.id]}` and `intervalsLoading={intervalsLoading[ex.id]}` props.

### 5i. Persist stroke to DB
